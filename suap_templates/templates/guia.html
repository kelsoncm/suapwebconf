Este guia de desenvolvimento compreende uma lista de regras que orientam a criação de páginas e funcionalidades para o SUAP, assim, tem o propósito de fornecer as diretrizes (relativas a CSS3, HTML5 e a linguagem de template para Django para disposição de elementos de leiaute, tipografia, formulários, botões/links, tabelas, grids, navegação, entre outros) com o objetivo de manter uma identidade visual padrão para todos as páginas do sistema.

{% box "Codificação" %}

Padrões de Escrita

  • Utilizar 4 espaços ao invés de tabs;
  • Acrescentar em todas as strings o Unicode. Ex.: u'Texto'
  • Manter títulos e labels sempre com as iniciais maiúsculas;
  • Pontuar corretamente todas as mensagens direcionadas ao usuário;
  • O verbose_name de um campo no models deve possuir como primeiro atributo a string com o título do campo, exceto quando o tipo for ForeignKey. Ex.: nome = models.CharField(u'Nome')

Recomendações de Nomenclaturas

Recomendado Não Recomendado
Adicionar Cadastrar
Editar Atualizar
Remover Excluir, Apagar
Visualizar Ver, Abrir

Instruções de CSS e HTML

  • Não utilizar o atributo style em tags HTML;
  • Não adicionar CSS ou Javascript diretamente no código HTML;
  • Evitar utilizar as seguintes tags br e hr;
  • Não utilizar as seguintes tags center, font e u, pois todas encontram-se depreciadas no HTML5;
{% endbox %}

Títulos

Toda página deve possuir apenas uma tag h2, que corresponde ao título da página. Visto que a tag h1 corresponde ao título do sistema, e deve ser encontrada via template, no topo.

Para montar uma hierarquia de títulos deve-se trabalhar hierarquicamente com h3, h4, em diante.

A barra de ações deve ser utilizada ao lado do título da página (h2), conforme o código mostrado ao lado:

  • Success

  • Default
  • Options
    • suboption 1
    • suboption 2
    • suboption 3

Regras
  1. Para mostrar o conteúdo do link dentro de um popup, deve ser adicionada à tag a a classe popup.
  2. Para botões que exigam uma confirmação (confirm javascript) antes de realizar a ação, basta adicionar a classe confirm ao link. Caso queira padronizar a mensagem do confirm, adicione como valor do atributo "data-confirm" na tag a.
  3. Para lista de ações dentro de uma célula de uma tabela que já possui conteúdo, deve ser adicionada à tag ul a classe float-right.
  4. Para marcar o botão como "checado", basta adicionar à tag a a classe checked.

  3. É possível inserir a classe disabled em uma tag tr caso deseja que toda a linha da tabela seja desabilitada visualmente.

<script src="/static/djtools/highcharts/highcharts.js"></script>

O código seguinte também é necessário para renderização do gráfico dentro do {% block content %}:

<script>
{% grafico|safe %}

{% box "Âncoras" %}

As âncoras são links relativos à áreas dentro da página atual. Quando clicadas, a página deve se mover e mostrar o tópico definido.

Na lateral direita desta página, é possível visualizar um exemplo de âncora fixa.

Código HTML

<ul class="ancoras">
    <li><a href="#id_1">Caixa de Entrada</a>
        <ul>
            <li><a href="#id_2">Aguardando recebimento</a></li>
            <li><a href="#id_3">Aguardando encaminhamento</a></li>
        </ul>
    </li>
    <li><a href="#id_4">Caixa de Saída</a></li>
</ul>

É necessário adicionar aos conteúdos da página a classe com_ancora para que eles se adaptem às âncoras.

{% endbox %}
{% box "Barra de Progresso" %}

Devem ser utilizadas para demonstrar visualmente o progresso de alguma ação dentro do sistema.

100%

25%

43.5%

3/6

Código HTML

<div class="progress">
    <p>25%</p>
</div>

Colocando o valor em porcentagem dentro do p, automaticamente a barra de progresso se expandirá para a largura determinada e receberá o title indicando o valor concluído.

É possível adicionar as classes alert, error, primary e success no div.progress para customizar as cores da barra de acordo com a situação do objeto referenciado.
